REST (Representational State Transfer) is a set of constraints for designing networked APIs. Instead of inventing custom verbs, you use HTTP methods (GET, POST, PUT, PATCH, DELETE) and descriptive URLs to communicate intent.

Design Checklist

  • Identify resources first (e.g., /posts, /posts/:id/comments).
  • Rely on HTTP status codes to communicate success and errors.
  • Return consistent JSON shapes and include hyperlinks (HATEOAS) when useful.
  • Document versioning and authentication early.

Example Request

GET /posts/42
Accept: application/json

A clean JSON response with the post body, author, and links to related resources makes it easy for clients to navigate your API without surprises.
